# 需求与边界（精简版）

## 业务目标
- 客户端（Electron）与服务端保持 MD5 集合一致
- 服务端只增不减，最终为并集

## 核心需求
- 双向同步：客户端推送缺失，客户端可拉取缺失
- 差异检测：支持分批与分页
- 高性能：4-5 万 MD5/用户，~10 并发
- 安全：固定 API 密钥 + `userKey` 白名单 + 权限/限流

## 非功能性
- 幂等：重复提交不产生副作用
- 可观测：统一日志与错误响应
- 可扩展：索引、缓存、布隆过滤器等优化

## 明确边界
- 仅同步 MD5 值（非原始敏感数据）
- 客户端负责分批、重试与断点续传
- 不提供通用用户体系，仅基于 `userKey` 白名单
